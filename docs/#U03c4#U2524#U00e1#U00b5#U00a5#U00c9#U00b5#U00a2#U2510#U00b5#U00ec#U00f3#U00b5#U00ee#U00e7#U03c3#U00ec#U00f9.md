# 素材替换指南

这份文档说明图片、视频、音频应该放在哪里，以及如何在 `js/config.js` 中接入页面。

---

## 1. 素材目录

```text
assets/
├── images/
│   ├── paper/             # 论文图、技术根因图、挑战机遇图
│   ├── architecture/      # 架构图、三种基础架构对比图
│   ├── demo/              # Demo 图，例如自然打断、用户附和、模型附和
│   ├── avatar/            # 数字人、机器人、Shennie 图
│   ├── charts/            # 雷达图、UTMOS 图、实验图
│   └── misc/              # 其他图片
├── videos/                # 视频素材
├── audio/                 # 音频素材
└── docs/                  # PDF 原文件或补充材料
```

---

## 2. 如何替换首页 Demo 图

第一步，把图片放到：

```text
assets/images/demo/natural_interrupt.png
```

第二步，打开：

```text
js/config.js
```

找到：

```js
heroDemo: {
  title: "自然打断：用户改口，系统立即调整",
  src: "assets/images/demo/natural_interrupt.png",
}
```

如果你的文件名不同，就改 `src`。

---

## 3. 如何替换雷达图

建议放到：

```text
assets/images/charts/result_radar.png
```

然后在 `js/config.js` 找到：

```js
resultRadar: {
  title: "多维能力对比雷达图",
  src: "assets/images/charts/result_radar.png",
}
```

如果图片较大，页面会自动 `object-fit: contain`，不会裁掉图中文字。

## 6. 如何替换架构图

### 三种基础架构对比图

建议路径：

```text
assets/images/architecture/architecture_compare.png
```

对应配置：

```js
architectureCompare
```

### Lychee-FD 主架构图

如果是 SVG，建议路径：

```text
assets/images/architecture/lychee_fd_architecture.svg
```

对应配置：

```js
fdArchitecture
```

SVG 可以直接展示，不要手动改 SVG 文件内容，避免破坏图形。

## 7. 如何接入 PDF 转出的 PNG 图

PDF 原文件可以先放到：

```text
assets/docs/
```

但网页展示不建议直接嵌入 PDF。推荐流程：

1. 用 PowerPoint、Adobe Acrobat、浏览器截图、`pdftoppm` 等方式把 PDF 导出为 PNG 或 SVG。
2. 放入对应图片目录，例如：

```text
assets/images/paper/modality_layer_conflict.png
```

3. 打开 `js/config.js`，修改对应 `src`。

例如模态层数矛盾图：

```js
modalityConflict: {
  src: "assets/images/paper/modality_layer_conflict.png"
}
```

## 8. 如何新增一张图片卡片

第一步，把图片放到合适目录，例如：

```text
assets/images/paper/new_technical_figure.png
```

第二步，在 `js/config.js` 的 `imageAssets` 中新增：

```js
newTechnicalFigure: {
  id: "newTechnicalFigure",
  title: "新的技术图",
  src: "assets/images/paper/new_technical_figure.png",
  type: "image",
  section: "技术根因",
  tags: ["技术图", "补充说明"],
  enabled: true,
  description: "这里写 1 到 2 句话说明。",
}
```

第三步，把它加入某个分组，例如：

```js
assetGroups: {
  methodEffectiveness: ["gradientHeatmap", "newTechnicalFigure"]
}
```

页面对应分组就会显示它。

## 9. 命名建议

建议使用英文或拼音命名，避免空格和特殊符号：

```text
natural_interrupt.png
user_backchannel.png
model_backchannel.png
result_radar.png
utmos_naturalness.png
avatar_speaking.png
```

如果必须使用中文文件名，也可以，但在 GitHub、服务器或不同系统之间更容易出现编码问题。
