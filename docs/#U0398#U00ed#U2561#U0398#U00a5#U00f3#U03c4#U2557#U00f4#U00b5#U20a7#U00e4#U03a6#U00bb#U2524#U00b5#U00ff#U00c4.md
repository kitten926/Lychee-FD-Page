# 页面结构说明

这份文档说明每个页面章节负责展示什么，以及对应的 HTML、CSS 和配置位置。

---

## 1. 文件分工

```text
index.html       页面结构和挂载点
css/styles.css   全站视觉样式和响应式适配
js/config.js     全站配置数据
js/main.js       页面渲染和交互逻辑
assets/          图片、视频、音频、PDF 等素材
docs/            中文说明文档
```

建议：日常新增素材和 case，主要改 `js/config.js`，不要频繁改 `index.html`。

## 3. HTML 挂载点说明

页面里有几类常见挂载点：

### 单张素材

```html
<div data-asset-id="heroDemo" data-asset-variant="hero"></div>
```

含义：从 `js/config.js` 的 `imageAssets.heroDemo` 读取图片并渲染。

### 一组素材

```html
<div data-asset-list="results" data-asset-variant="chart"></div>
```

含义：从 `assetGroups.results` 读取一组图片 id，再到 `imageAssets` 中取配置。

### 指标卡片

```html
<div data-metrics></div>
```

含义：从 `metrics` 渲染四个大指标。

## 4. CSS 样式在哪里

所有样式集中在：

```text
css/styles.css
```

常见样式块：

```text
:root                  颜色变量和尺寸变量
.site-header           顶部导航
.hero                  首页首屏
.metric-card           指标卡片
.asset-card            图片/视频统一玻璃卡片
.demo-card             Demo case 卡片
.lightbox              图片点击放大预览
@media                 移动端适配
```

如果只想调整颜色，优先改 `:root` 变量。

---

## 5. JS 逻辑在哪里

```text
js/main.js
```

主要函数：

```text
renderNavigation       渲染导航
renderMetrics          渲染指标卡
renderHighlights       渲染技术亮点
renderAssets           渲染图片/视频素材卡
renderDemoTabs         渲染 Demo tab 和 case 卡
renderPackageSpec      渲染素材规范字段
enableLightbox         图片点击放大
enableActiveNav        导航当前章节高亮
animateMetricCounters  指标递增动画
```

日常维护一般不用改 `main.js`，除非你要新增交互形式。

---

## 6. 后续扩展建议

1. 如果新增一个完整章节，先在 `index.html` 添加 section，再在 `navigation` 中添加导航项。
2. 如果只是新增图片，优先改 `imageAssets` 和 `assetGroups`。
3. 如果新增 Demo，优先改 `demoCases`。
4. 如果新增新的 Demo tab，先改 `demoTabs`，再新增对应 `category` 的 case。
5. 如果新增真实音频播放器，可以在 `main.js` 的 `materialButton` 附近扩展为内嵌 `<audio controls>`。
