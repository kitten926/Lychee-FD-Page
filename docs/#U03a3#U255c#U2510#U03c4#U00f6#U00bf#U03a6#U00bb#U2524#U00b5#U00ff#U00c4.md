# 使用说明

这份文档说明如何打开页面、修改基础文案、修改导航、调整颜色，以及如何找到对应页面章节。

---

## 1. 如何打开页面

### 方式一：直接双击

在项目根目录双击：

```text
index.html
```

适合快速查看页面结构。

### 方式二：本地服务预览，推荐

在项目根目录打开终端，运行：

```bash
python -m http.server 8080
```

浏览器打开：

```text
http://127.0.0.1:8080
```

如果你后续接入了音频、视频、SVG，推荐都用这种方式预览。

---

## 2. 如何修改标题和首页文案

打开：

```text
js/config.js
```

找到：

```js
site: {
  title: "Lychee-FD",
  subtitle: "全双工大模型驱动的具身类人交互",
  slogan: "从轮流说话，到自然共话。",
  description: "让 AI 在真实对话中同时倾听、表达、理解打断与生成反馈，从“轮流说话”走向“自然共话”。",
}
```

直接改这里即可。

---

## 3. 如何修改导航

打开：

```text
js/config.js
```

找到：

```js
navigation: [
  { label: "背景", target: "background" },
  { label: "挑战", target: "challenge" },
]
```

`label` 是导航显示文字。  
`target` 是页面章节的 `id`，需要和 `index.html` 里的 section id 对上。

例如：

```html
<section class="section" id="background">
```

对应配置：

```js
{ label: "背景", target: "background" }
```

---

## 4. 如何调整颜色

打开：

```text
css/styles.css
```

最上方有颜色变量：

```css
:root {
  --page-bg: #f5f8ff;
  --ice: #1d8eff;
  --lavender: #9c7cff;
  --text: #102033;
}
```

建议优先改这些变量，不要到处手动改单个颜色。

常用变量含义：

```text
--page-bg        页面浅色背景
--paper          玻璃卡片背景
--text           正文颜色
--heading        标题颜色
--muted          辅助文字颜色
--ice            冰蓝主色
--lavender       极光薰衣草点缀色
--green          荔枝叶绿色点缀色
--orange         控制通道橙金色
--line           卡片边框颜色
--shadow         卡片阴影
```

## 5. 如何找到对应页面章节

页面主体在：

```text
index.html
```

每个章节都有注释和 id，例如：

```html
<!-- 04 技术根因 -->
<section class="section" id="root-cause">
```

主要章节：

```text
01 首页首屏             id="hero"
02 项目背景             id="background"
03 三大核心挑战         id="challenge"
04 技术根因             id="root-cause"
05 Lychee-FD 架构       id="architecture"
06 语义对齐通道         id="semantic-channel"
07 技术亮点             id="highlights"
08 Demo 展示区          id="demo"
09 性能结果             id="results"
10 数字人 / 机器人展示  id="embodied"
11 Case 素材采集规范    id="case-spec"
12 未来展望             id="outlook"
```

---

## 6. 哪些内容不要直接改 HTML

下面这些建议优先改 `js/config.js`：

```text
导航项
首页指标
技术亮点
图片资源路径
Demo case
数字人展示图片
性能结果图片
素材规范字段
```

`index.html` 更适合改页面结构，不适合频繁改数据。
